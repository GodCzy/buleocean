# 平台架构

## 定位

蓝海智询不是单页展示站，而是围绕水生动物疾病知识组织、检索、问答与辅助决策构建的完整知识平台。仓库需要同时承载前端展示、服务编排、知识入库、检索基础设施和运行配置。

## 分层

### 体验层

- 首页驾驶舱
- 主题分区与知识路径工作台
- 智能问答界面
- 知识图谱界面
- 向量数据库与评测界面

### 服务层

- FastAPI 接口
- 问答编排与 Agent 调度
- 检索请求编排
- 文件处理与任务执行

### 数据与知识层

- 病-药-环知识图谱
- 向量数据库与分片索引
- 业务元数据与用户数据
- 文档资产与对象存储

### 部署与运维层

- Docker Compose 运行栈
- 环境变量模板
- 构建脚本与验证脚本
- 测试、日志和健康检查

## 架构原则

- 平台层负责品牌、路由、共享布局、运行配置和公共能力。
- 领域层负责鱼类知识、疾病知识、图谱语义、问答模板和检索语义。
- 展示层允许前端先行完成系统叙事，真实模型和数据服务可逐步接入。
- 公开仓库优先保留完整系统感，同时避免暴露与蓝海智询无关的历史业务语义。

## 业务闭环

1. 汇集病例资料、养殖记录、疾病手册、检测指标和水质监测数据。
2. 通过知识抽取、分片、向量化和图谱建模形成统一知识底座。
3. 通过混合检索、图谱推理和多轮问诊生成可解释的处置建议。
4. 通过评测、反馈和知识库迭代不断提升回答质量与运营能力。
