# 前端结构

## 目标

前端不是简单展示截图，而是把蓝海智询的系统能力组织成一条可演示、可理解、可截图的产品路径。

## 页面主线

- `themes/*`：专题知识空间与模块入口
- `knowledge/*`：知识路径工作台，承接主题上下文
- `agent`：类 ChatGPT 的问诊式智能问答
- `graph`：病-药-环知识图谱与实体关系展示
- `database`：向量索引、检索评测与知识库管理展示

## 共享状态

- 主题上下文：跨主题页、问答页、图谱页同步当前知识域
- 知识路径上下文：维护工作台中的问题起点、分支状态和 handoff 信息
- 业务信息配置：驱动首页、模块卡片、入口链接和展示文案

## 组件组织

- 通用组件：导航、消息面板、图表、信息卡片、列表与弹窗
- 蓝海智询专题组件：图谱演示、问答演示、向量检索演示
- 知识路径组件：分支画布、证据栏、节点详情和下一步建议

## 前端设计原则

- 优先让评委一眼看出“这是一个专业系统”，而不是文案堆叠页。
- 图谱、问答、数据库三个核心界面要有明显差异化。
- 页面文案服务于鱼类疾病知识、RAG、向量检索和风险预警，不再沿用旧项目语义。
- 即使后端能力未全部实装，前端也要完整表达真实系统形态。
