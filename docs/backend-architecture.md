# 后端结构

## 目标

后端负责把知识问答平台的接口、知识服务、任务执行和系统管理组织成可运行的工程骨架。公开仓库保留这些结构，是为了让项目看起来像完整系统，而不是只有展示前端。

## 服务分区

### 基础服务

- `system`：系统信息、健康检查、配置读取
- `auth`：登录、令牌、用户身份

### 知识与问答

- `chat`：问答会话、消息、模型调用编排
- `knowledge`：知识库、文件、分片、检索配置
- `graph`：图谱查询、关系浏览、知识节点接口
- `evaluation`：检索评测与问答效果评估

### 扩展能力

- `mindmap`：结构化摘要与知识脉络生成
- `tasks`：异步任务与后台执行
- `skills`、`tools`、`mcp`：外部能力接入与工具管理

## 设计原则

- 保持 API 入口稳定，方便前端界面与后续真实服务接入。
- 把知识层、图谱层、检索层与系统层职责拆开，避免单模块强耦合。
- 允许前端先行演示，后端逐步从静态编排切换到真实数据与模型服务。

## 公开仓库策略

- 保留后端目录、依赖、服务入口和部署配置，体现工程完整性。
- 对明显暴露旧项目身份的业务样例、历史文档和遗留专题做裁剪。
- 让仓库对外呈现为蓝海智询的知识问答平台，而不是无关领域的二次包装。
