# 模块扩展

## 目标

蓝海智询需要支持多个知识专题，而不是把所有能力堆在一个页面里。模块扩展文档描述的是：如何在统一平台下继续接入新的知识主题、检索视图和问答场景。

## 当前模块

- `fish-atlas`：鱼类知识图谱
- `rag-lab`：RAG 检索工作台
- `vector-vault`：向量索引中心

## 平台层负责

- 品牌与导航
- 路由与页面入口
- 共享布局与状态同步
- 通用问答、图谱、数据库界面

## 模块层负责

- 主题元数据
- 领域文案与展示语义
- 图谱节点关系与推荐逻辑
- 专题默认问题与证据表达

## 接入约束

新模块至少需要提供：

- 主题基础信息：名称、副标题、简介、入口路由、标签、亮点
- 知识路径默认问题与工作台上下文
- 主题详情页展示素材
- 图谱或问答至少一个可视化重点

## 目标效果

- `/themes` 可以发现模块
- `/knowledge/<theme>` 可以承接模块上下文
- `/agent` 与 `/graph` 可以消费模块传入的领域语义
- 模块扩展不需要重写整个平台骨架
